# Roadmap

## Fase 1. Diseño del flujo

- definir ramas y estrategia de release;
- definir políticas mínimas de merge;
- inventariar secretos y entornos.

## Fase 2. Seguridad en CI

- lint, test y build;
- escaneo de dependencias;
- detección de secretos;
- generación de reportes.

## Fase 3. Entrega controlada

- despliegue a staging;
- smoke tests;
- aprobación antes de producción;
- rollback documentado.

## Fase 4. Evidencia de operación

- métricas del pipeline;
- logs de release;
- matriz de riesgos;
- README final orientado a reclutadores y pares técnicos.
