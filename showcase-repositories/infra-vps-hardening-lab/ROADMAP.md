# Roadmap

## Fase 1. Descubrimiento

- definir distribución Linux objetivo;
- establecer superficie de ataque inicial;
- documentar puertos, servicios y dependencias.

## Fase 2. Baseline seguro

- hardening de SSH;
- firewall por defecto en deny;
- usuarios no privilegiados;
- actualizaciones y paquetes mínimos.

## Fase 3. Publicación segura

- `Nginx` como reverse proxy;
- TLS activo;
- headers y configuración base defensiva;
- logs centralizados localmente.

## Fase 4. Evidencia y documentación

- checklist de validación;
- tabla de riesgos;
- runbook de operación;
- README publicable para portafolio.
